## 확인 질문 답변

| 질문 | 답변 |
|---|---|
| 현재 작업 폴더의 전체 경로는 무엇인가? | `pwd` 명령으로 확인한 전체 경로를 기록한다. 예: `/Users/사용자이름/cloud-native-week1` |
| 작업 폴더를 홈 폴더에 만들었는가, Desktop에 만들었는가? | 둘 중 선택한 위치를 기록한다. 홈 폴더(`~`)와 Desktop(`~/Desktop`)은 다른 위치이며, Desktop은 Finder에서 찾기 쉽지만 iCloud 동기화 주의가 필요하다. |
| 오늘 만든 확인 기록 중 공유 가능한 것과 보호해야 할 것은 무엇인가? | 공유 가능: 작업 폴더 경로, `pwd` 출력, 명령과 실행 결과. 보호해야 할 것: 비밀번호, 토큰, MFA 코드, 인증 코드. |
| 다른 사람이 같은 폴더 구조를 만들려면 어떤 명령이 필요한가? | `cd ~`, `mkdir -p cloud-native-week1`, `cd cloud-native-week1`, `pwd` 순서로 실행하면 된다. 경로 기록을 남기면 재현이 가능하다. |
| 폴더 이름은 아무래도 상관없는가? | 이름은 자유지만 수업 확인 기록에서 같은 이름을 쓰면 재현성이 좋아진다. |
| Desktop에 만들면 터미널에서 찾을 수 없는가? | 아니다. `cd ~/Desktop`으로 이동하면 Desktop 아래 폴더도 터미널에서 사용할 수 있다. |
| 막힘 기록은 실패 기록이라 감점이 되는가? | 아니다. 막힘 기록은 운영에서 중요한 상태 보고다. 단, 증상과 시도한 확인을 같이 써야 한다. |

## notes

### 작업공간의 운영 의미

작업공간은 단순한 폴더가 아니다. 현업에서는 장애나 배포 실패가 발생했을 때 "어떤 directory에서 어떤 명령을 실행했는가"가 중요한 재현 조건이 된다. 같은 명령도 다른 경로에서 실행하면 다른 파일을 읽거나, 다른 설정을 사용하거나, 아무 파일도 찾지 못할 수 있다.

### 작업 폴더 만들기

```bash
pwd
cd ~
mkdir -p cloud-native-week1
cd cloud-native-week1
pwd
```

`mkdir -p`는 같은 폴더가 이미 있어도 오류 없이 지나가므로 수업 중 재실행해도 안전하다.

Desktop에 만드는 경우:

```bash
pwd
cd ~/Desktop
mkdir -p cloud-native-week1
cd cloud-native-week1
pwd
```

### 확인 기록 판단 기준

| 판단 질문 | 강한 확인 기록 | 약한 확인 기록 |
|---|---|---|
| 어디에서 실행했는가? | 전체 경로와 명령이 함께 있음 | 폴더 이름만 기억함 |
| 무엇을 남겼는가? | 출력을 README 또는 기록에 요약 | 화면을 봤다고만 적음 |
| 공개 가능한가? | 비밀값을 제외한 경로/출력 | 토큰이나 인증코드가 섞인 캡처 |

### Day2 → 이후 주차 연결

컨테이너 실행 환경의 작업 디렉터리, Kubernetes manifest의 mount 경로, Terraform module 경로는 모두 "어디에서 실행되는가"라는 질문으로 돌아온다. 오늘 기록한 경로 확인 기록이 그 기본 언어다.

### 확인 기록 흐름

```mermaid
flowchart LR
    A[Day1 개인 목표] --> B[작업 폴더 경로]
    B --> C[실행 명령]
    C --> D[확인 가능한 출력]
    D --> E[README 확인 기록]
```

### 핵심 문장

```text
"안 됩니다"보다 "어느 경로에서 어떤 명령을 실행했고 어떤 출력이 나왔습니다"가
훨씬 좋은 보고다. 이 습관이 incident report, runbook, RCA의 출발점이다.
```
