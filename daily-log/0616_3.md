**첫 번째, PostgreSQL 실습**

DB 설치야 해봤지만, 오늘은 그 설치가 OS에 정확히 무엇을 남기는지를 의식하면서 본 게 달랐다.

실행 파일 + 백그라운드 서비스 + 포트 + 계정 + 데이터 폴더 + 설정 파일 — 이게 한 번에 등록된다는 걸 알면서도 딱 정리해서 생각해본 적은 없었는데, 오늘 명확하게 정리됐다.

`connection refused`랑 `password authentication failed`를 각각 어떤 조건에서 만나는지 분리해서 보니까, 막혔을 때 어디부터 봐야 하는지도 훨씬 깔끔하게 정리된 느낌 😅

**두 번째, 포트 충돌이랑 .env 관리**

예전에 두 프로젝트 진행 기간이 겹쳤던 적이 있는데, 프로젝트 A는 로컬 DB, 프로젝트 B는 Docker DB라 애초에 같은 포트를 쓸 수밖에 없는 구조였다. 그래서 번갈아 쓸 때마다 기존 프로세스 종료하고 새로 띄우는 걸 반복했는데, **Docker port binding**으로 `5433:5432` 이런 식으로 host 포트만 다르게 잡으면 둘을 동시에 띄울 수 있었겠구나...라는 생각이 번뜩...🫠

`.env` 환경별 분리도 쓰고 있었는데, 로컬이랑 production이 섞였을 때 운영 DB에 붙는 상황... 알고는 있었지만 오늘 다시 보니까 경각심이 새로 생겼다. (아이 무서워...😱)
