**첫 번째, VM vs Container 자원 할당 방식**

VM은 자원을 미리 딱딱 나눠서 고정 할당하고, 다른 VM이 놀고 있어도 그 자원은 묶여있다는 게 인상 깊었다.

Container는 반대로 host 자원을 동적으로 공유한다. 바쁜 container가 더 쓰고, 끝나면 반환.

알고는 있었는데 이렇게 명확하게 대비해서 설명을 들으니까 "아 그래서 container가 가볍다는 거였구나"가 제대로 정리됐다 😅

**두 번째, PostgreSQL 16이랑 18 동시에 띄우기**

container port는 둘 다 5432인데 host port만 15432, 15433으로 다르게 잡으면 동시에 접속할 수 있다는 걸 직접 실습으로 확인했다.

저번에 프로젝트 두 개 포트 충돌 때문에 번갈아 껐다 켰다 했던 기억이 있는데... 그때 이걸 알았더라면 😩

의도적으로 port 충돌을 만드는 실습도 했는데, 실패해도 container가 Created 상태로 남는다는 게 새로웠다.

이 부분 실습에서 `\` 줄바꿈 실수로 명령어 한 번 날려먹은 게 좀 아쉬웠지만... 덕분에 `\` 앞에 공백 있어야 한다는 건 절대 안 잊을 것 같다 😅

**세 번째, DB timezone은 UTC로 두는 이유**

예전에 DB timezone을 KST로 설정하면 됐지 않나? 싶었는데, restart하면 이후 데이터가 UTC로 찍혀서 9시간 차이가 생긴다는 실제 사례를 들으니까 바로 이해됐다.

DB는 UTC, 표시 전환은 애플리케이션에서. 이거 꼭 기억해둬야지 🗒️
