[클라우드 네이티브 3회차] 0619_6 🐥

오늘은 3-tier 구조를 container로 구성하고, network 분리랑 volume 지속성을 증거로 남기는 실습을 했다.

실습했던 거를 끄적여보면,✏️

### **첫 번째, 3-tier를 container로 직접 만들어보니 network가 핵심이었다**

frontend, backend, DB를 각각 별도 container로 띄우고 network를 두 개 나눠서 연결했다.

- DB: back-net 연결
- backend: run할 때 back-net 연결 → 이후 `docker network connect`로 front-net 추가 연결
- frontend: run할 때 front-net 연결

결국 back-net에는 backend + DB만, front-net에는 frontend + backend만 있는 구조다.

여기서 중요한 건 **frontend를 back-net에 연결하지 않은 것**이다. frontend가 back-net에 들어가면 DB에 직접 접근이 가능해진다. network 분리의 목적 자체가 사라지는 것. frontend는 front-net을 통해 backend랑만 통신하고, DB는 반드시 backend를 거쳐서만 접근하게 설계한 거다 🗒️

### **두 번째, network 분리가 실제로 막는다는 걸 직접 확인했다**

`docker exec paperclip-weekend-front ping -c 2 paperclip-weekend-db || true`를 실행했더니 `bad address 'paperclip-weekend-db'`가 나왔다.

![ping-front-db-fail](assets/0619_6/ping-front-db-fail.png)

이름조차 못 찾는다는 게 포인트였다. 같은 network에 없으면 Docker DNS가 이름을 해석해주지 않는다. 그냥 "연결은 되는데 막힌다"가 아니라 "존재 자체를 모른다"는 것. 이게 오히려 더 강력한 격리인 것 같았다 😮

### **세 번째, volume이 있으면 환경변수 초기화는 무시된다**

DB container를 삭제하고 `POSTGRES_DB=weekend_changed`로 이름을 바꿔서 같은 volume으로 다시 실행했는데, DB 이름이 바뀌지 않았다.

이유는 PostgreSQL이 volume에 데이터가 이미 있으면 `POSTGRES_DB`, `POSTGRES_PASSWORD` 같은 초기화 환경변수를 무시하기 때문이다. `docker logs`에서 `Skipping initialization` 메시지로 확인했다. 환경변수는 볼륨이 완전히 비어있을 때(최초 기동 시)에만 적용된다 🗒️

![logs-db-skip](assets/0619_6/logs-db-skip.png)

### **네 번째, base image 하나로 이미지 크기가 몇 배씩 차이 난다**

강사님께서 주신 `measure-build.sh` 스크립트로 base image별 빌드 시간이랑 크기를 비교했다.

| target | base image | 빌드 시간 | 이미지 크기 |
|--------|-----------|---------|-----------|
| frontend | `nginx:stable` | 3.9s | 256MB |
| frontend | `nginx:stable-alpine` | 0.5s | 101MB |
| frontend | `nginx:stable-trixie` | 0.5s | 256MB |
| backend | `node:22` | 14.0s | 1.62GB |
| backend | `node:22-slim` | 5.7s | 348MB |
| backend | `node:22-alpine` | 1.3s | 229MB |

nginx:stable-alpine이 nginx:stable보다 약 2.5배 작고, node:22-alpine은 node:22보다 무려 약 7배 작다. 빌드 시간도 alpine이 훨씬 빠르다.

이미지가 크면 push/pull 시간이 늘고, 새 서버에 배포할 때도 느려지고, 취약점도 더 많이 포함될 수 있다. alpine은 패키지를 최소한만 담아서 취약점이 숨어있을 패키지 수 자체가 적다. alpine을 쓰는 게 기본값이 되어야 할 것 같다 🗒️
